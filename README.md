# Nak-AI (仲居)

配信者の「第二の脳」として機能する、配信者専用リアルタイムAIアシスタントソフトウェア。

## 概要

日本語の「仲居（なかい）」から名付けられた **Nak-AI** は、配信者のマイク音声と配信サイトのコメントをリアルタイムに分析し、配信者のゲーム実況やリスナーとのコミュニケーションを滑らかにするための配信者専用AIアシスタントソフトウェアです。

配信中の沈黙（実況の途切れ）時に直前の話題を思い出させたり、配信でコメントしたリスナーの過去のコンテキストを提示したりすることで、配信者の実況およびコミュニケーションを補助します。

---

## システムの仕組み

```
+------------------+
|  マイク音声入力  | ----+
+------------------+     |
                         v
+------------------+   +------------+   +------------------+
|  コメント欄入力  | ->|   Nak-AI   | ->|  配信者向け画面  |
|  (配信サイト)    |   +------------+   | (支援テキスト等) |
+------------------+                    +------------------+
```

Nak-AI は以下の2つの入力をリアルタイムに監視・分析し、配信者を補助するための情報をテキストとして画面に提示します。

1. **配信者の音声（インプット1）**
   - 配信者のマイク音声をリアルタイムで文字起こしし、直前の話題や進行状況を追跡します。
2. **配信サイトのコメント（インプット2）**
   - リスナーから送られてくるコメントを取得し、過去のインタラクション履歴と照合します。

---

## 主な機能

### 1. 実況復帰アシスタント（実況の思い出させ・コンテキスト補助）
配信初挑戦のゲームに集中して無言になった際や、次に何を話すか見失ってしまった際、直前まで話していた内容（コンテキスト）を忘れないよう補助します。具体的なトークテーマの提案や指示はせず、直前の会話文に基づいた「その後どうなりましたか？」とシンプルに実況を復帰させるメッセージを提示します。

> 💡 **表示例**
> - 「先程まで『（直前の話題）』について話していたようですが、その後どうなりましたか？」

### 2. リスナーコンテキストの提示
リスナーがその配信で初めてコメントを送信した際、過去の配信におけるそのリスナーのコメント履歴や出来事のサマリーを配信者の画面に瞬時に提示します。これにより、配信者が「このリスナーとは過去にどんなやり取りをしたか」を思い出す手助けをします。※提示された情報を元に、どのように声をかけるか（スルーするか、話題に触れるかなど）はすべて配信者の判断に委ねられます。

> 💡 **表示例**
> - **@listener_user**: 前回の配信で『明日テストがある』とコメントしていました。
> - **@new_viewer**: 前回の配信の終盤に初めてコメントしたユーザーです。

## 初期設定と APIキーの取得方法

Nak-AI の「リアルタイムAIアシスタント（誤字補正＆アドバイス）」機能を利用するには、**Google AI Studio** から発行された Gemini APIキーが必要です。

### 🔑 APIキーの取得手順

最新の情報や詳細な手順は、Google公式の [Gemini API クイックスタートガイド](https://ai.google.dev/gemini-api/docs/quickstart?hl=ja#before_you_begin) も併せてご参照ください。

1. **[Google AI Studio](https://aistudio.google.com/)** にアクセスします。
2. お持ちの Google アカウント（Gmail）でサインインします。
3. 画面左上の **「Get API key」** ボタンをクリックします。
4. **「Create API key」** ボタンをクリックします。
5. ポップアップが表示されるので、**「Create API key in new project」**（新しいプロジェクトでキーを作成）をクリックします。
6. 発行された APIキー（`AIzaSy...` で始まる文字列）をコピーします。
7. Nak-AI アプリケーションを起動し、**「LLM (Gemini) 連携設定を開く」** アコーディオンを展開して、入力欄に貼り付け **「保存」** をクリックします。

### 💰 料金と利用制限について

* **完全無料で利用可能 (Free Tier)**:
  - Gemini 2.5 Flash モデルは、無料枠が非常に充実しています。
  - **毎分15リクエスト / 1日1500リクエスト**まで完全無料で利用できます。
  - Nak-AIでは、発話の区切り（VAD）ごとにAPIを呼び出す設計になっているため、数時間の配信実況であっても、この無料枠の範囲内で十分に終日稼働させることが可能です！

## 開発環境と動作要件

本ソフトウェアは、以下の環境で開発・動作確認を行っています。
* **OS**: Windows
* **開発言語**: Python 3.13.5
* **開発アシスタント**: Antigravity (AI coding assistant)

### 🔔 通知音SEのセットアップ（開発者向け）

仲居助言の通知音に使用するSEファイルは、OtoLogicの利用規約（素材そのものの再配布禁止）に従い、リポジトリには含まれていません。
開発・ビルドを行う場合は、以下の手順でローカル環境にセットアップしてください。

**前提条件**: [ffmpeg](https://ffmpeg.org/download.html) がインストール済みで、PATHに追加されていること。

1. [OtoLogic](https://otologic.jp) から「**神楽鈴02**」のZIPファイルをダウンロード
2. 以下のコマンドを実行（ダウンロードフォルダのZIPを自動検出します）：

```bash
python scripts/download_se.py
```
