# 配信エンゲージメント・分析ツール調査報告（FrostyToolsおよび類似ツール）

## 概要

TwitchやYouTubeでの配信において、AI技術や自動化スクリプトを活用してリスナーのエンゲージメント（関係性・活発さ）を高めたり、配信の分析・コンテンツの自動作成を行ったりするツールが急速に増加しています。
本ドキュメントでは、AI搭載型のクラウドツール（**FrostyTools, Algochat.io, StreamChat AI**）および、ローカルPC環境で動作する既存の配信自動化・サポートツール（**Streamer.bot, SAMMI, ゆかりねっと, OBSスクリプト**）の合計7ツールについて、同一の主要4項目（**「音声キャプチャ」「利用LLM」「コンテキスト保持」「ユーザーへのフィードバック」**）を用いて評価・比較した結果をまとめます。

## 2. ローカル動作ツールの評価項目別の詳細

### ① 音声キャプチャ (Audio Capture)
* **Streamer.bot / SAMMI / OBSスクリプト**:
  - **現状**: 標準機能としては、マイク音声から「何を喋っているか（言葉）」をリアルタイムに認識する機能はありません。OBSスクリプトやStreamer.botは「現在の音声の音量レベル（入力があるかないか、ミュートか否か）」の検知に留まります。
  - **拡張性**: C# (Streamer.bot) や Python (OBSスクリプト) を記述できるため、WindowsのサウンドAPIや外部のオーディオキャプチャライブラリを統合し、音声認識エンジン（ローカルWhisper等）と連携させることは技術的に可能です。
* **ゆかりねっと**:
  - **現状**: ローカルの音声ミキサーまたはOSのマイク入力デバイスを常時監視し、音声をキャプチャします。キャプチャしたデータは、Google Chromeブラウザのリモート音声認識機能や、ローカルに接続された認識API（UDトーク等）に渡してテキスト化します。

### ② 利用しているLLM (LLM Used)
* **ローカル自動化ツール全般**:
  - **現状**: 標準機能としてLLMを内蔵していません。
  - **拡張性**: すべてのローカルツールは「外部へのHTTPリクエスト（GET/POST）」や「スクリプト（C#, JavaScript, Python, Lua）」を実行できるため、**OpenAI, Claude, Gemini などの外部APIをローカルから呼び出すこと**ができます。また、ローカルPC上で動作させた **Local LLM (LM StudioやOllamaなどを介したLlama 3等)** と通信させて完全ローカルのAIアシスタントを構築することも可能です。

### ③ コンテキストの保持 (Context Retention)
* **ローカル自動化ツール (Streamer.bot, SAMMI, OBSスクリプト)**:
  - **一時メモリ**: メモリ上のグローバル変数（Global Variables）として一時的にデータを保持します。「前回のサブスクライバーの名前」「今日の累計コメント数」などをセッション内で追跡するのに適しています。
  - **永続化**: ローカルPC内のハードディスク上に、テキストファイル、JSONファイル、またはローカルにセットアップした SQLite などのデータベースファイルとしてデータを書き出し、保存・クエリします。
* **ゆかりねっと**:
  - **現状**: コンテキストの保持（過去の会話履歴を追跡・学習するなど）は行いません。入力された音声をその場でテキスト化し、音声合成に流して完結する「ステートレス」な処理を行います。

### ④ ユーザーへのフィードバック (User Feedback)
* **ローカルツール特有の強力なフィードバック手段**:
  - クラウドツール（チャット欄への自動書き込みがメイン）と異なり、ローカル動作するツールは **「PC上のほぼすべてのリソース」** をフィードバック先として制御できます。
  - **OBSの制御**: WebSockets経由で、配信画面上の画像や動画ソースの表示/非表示、カラーフィルターの適用、3D回転、シーンの自動切り替えなど、視覚的にリッチな演出を実行します。
  - **ローカルメディア/TTS再生**: 配信者PCのスピーカーから効果音（SE）を鳴らしたり、ローカルの音声合成エンジン（VOICEROID等）から声を出力したりします。
  - **チャット・Webフック連携**: 配信プラットフォームのチャット欄へのメッセージ送信、Discordへの通知自動投稿などを実行します。

## 3. 総括的な技術分析

1. **AI/LLMの処理特性**:
   - クラウド/AIツール（前半3つ）は、インプットに対して「LLMが知的な返答や要約を作成すること」に最適化されています。
   - 一方、ローカルツール（後半4つ）は、インプットに対して「OBSの画面を動かしたり音を鳴らしたりする物理的な演出制御（アクション）」に最適化されています。
2. **音声認識のローカル実装の難易度**:
   - `ゆかりねっと` を除き、多くのローカル自動化ツールにおいて「マイク音声のリアルタイム認識・テキスト化（STT）」は標準装備されておらず、外部モジュールとの連携が必要です。これは、音声入力の処理およびSTTがローカル開発において比較的ハードルの高い技術領域であることを示しています。
3. **データセキュリティと拡張性**:
   - ローカルツールは、すべてのコンテキスト（変数やJSONファイル）が配信者のローカルPC内に保存されるため、データの漏洩リスクが極めて低く、配信者が自身のデータを完全に掌握できるという大きな強みを持っています。
