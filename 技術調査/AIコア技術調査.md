# AIコア技術調査報告（Gemini, GPT-4o-mini, Faster-Whisper）

## 概要

本ドキュメントでは、配信者支援ソフトウェアの開発においてコアとなるAI技術（LLMおよび音声認識：STT）の3つの候補について、**「特徴」「利用方法」「料金・コスト」**を調査・比較した結果をまとめます。
また、エンドユーザー（配信者個人）が自身のアカウントでこれらのAPIを利用するための**「認証・APIキーの運用方法」**についての検討結果を追記しました。

調査対象技術：
1. **Gemini 1.5 Flash / 2.0 Flash** (Googleの超高コスパ・大容量コンテキストLLM)
2. **GPT-4o mini** (OpenAIの高速・格安軽量LLM)
3. **Faster-Whisper** (ローカルで超高速動作するオープンソース音声認識ライブラリ)

---

## 1. Gemini 1.5 Flash / 2.0 Flash

Googleが提供する、速度とコストパフォーマンス、そして超巨大な「コンテキスト窓（受け入れられるテキスト量）」に特化したモデルです。

### ① 特徴
* **圧倒的なコンテキスト窓**: 最大 **100万〜200万トークン** を一度に入力可能（数時間の音声データや、膨大なコメント履歴をそのまま入力できます）。
* **マルチモーダル**: テキストだけでなく、音声ファイルを直接入力して理解させることが可能（STTを通さずに、マイクの録音ファイルをそのまま流すアプローチも可能です）。
* **リアルタイムAPI**: 低遅延の双方向WebSocket通信に対応しており、リアルタイムな音声対話アプリの構築に向いています。
* **高い開発者サポート**: `Structured Outputs`（JSON出力の強制）や `System Instructions`（システム指示）が非常に強力です。

### ② 利用方法 (Python例)
Google AI Studio から APIキー を取得し、SDK（`google-genai`）を利用します。

```python
import os
from google import genai
from google.genai import types

# クライアント初期化
client = genai.Client(api_key=os.environ.get("GEMINI_API_KEY"))

# 実行
response = client.models.generate_content(
    model='gemini-1.5-flash',
    contents='配信中の沈黙を検知しました。直前の実況内容「次はあのボスに挑戦します」を元に、短い問いかけを作ってください。',
    config=types.GenerateContentConfig(
        system_instruction="あなたは配信者を裏から支える優秀なアシスタント（仲居）です。",
        temperature=0.7,
    )
)

print(response.text)
```

### ③ 料金・コスト
* **Google AI Studio (無料枠 / Free Tier)**:
  - **完全無料**（一定の制限：15 RPM（回/分）、150万 TPD（トークン/日）などの枠内であれば無料で検証・開発可能）。
  - ※注意: 無料枠のデータはモデルの学習に利用される可能性があります。
* **有料枠 (Pay-as-you-go)**:
  - **入力トークン**: $0.075 / 100万トークン
  - **出力トークン**: $0.300 / 100万トークン
  - **コンテキストキャッシュ**: よく使う大規模コンテキスト（システムプロンプトや基本情報）をキャッシュしておくことで、入力コストを最大50%削減可能。

## 2. GPT-4o mini

OpenAIが提供する、GPT-3.5の後継となる超軽量・高速なLLMです。

### ① 特徴
* **クラス最高峰の応答速度**: 非常に軽量化されており、応答生成の遅延（レイテンシ）が極めて低いです。
* **構造化出力 (Structured Outputs)**: JSONスキーマを厳密に満たした出力を保証するため、プログラムが処理しやすいデータ形式で安定して応答を得られます。
* **長文対応**: 128,000トークンのコンテキスト窓を持ち、実用上十分な履歴を保持できます。

### ② 利用方法 (Python例)
OpenAI Platform から APIキー を取得し、`openai` ライブラリを利用します。

```python
import os
from openai import OpenAI

# クライアント初期化
client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

# 実行
completion = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "あなたは配信アシスタントです。配信者の無言を補助してください。"},
        {"role": "user", "content": "先程まで『マイクラで家を建てていた』ようですが、沈黙しました。声かけのヒントを1つ作ってください。"}
    ]
)

print(completion.choices[0].message.content)
```

### ③ 料金・コスト
* **無料枠**: なし（新規登録時の少額クレジットのみ）。
* **API使用料**:
  - **入力トークン**: $0.150 / 100万トークン
  - **出力トークン**: $0.600 / 100万トークン
  - **キャッシュされた入力**: $0.075 / 100万トークン（繰り返されるプロンプトに対して自動割引が適用）

## 3. Faster-Whisper

OpenAIが公開した高性能音声認識モデル「Whisper」を、C++ (CTranslate2) を用いて超高速化・軽量化したオープンソースライブラリです。

### ① 特徴
* **ローカル動作**: 配信者のローカルPC上で動作するため、**完全にオフライン・無料で利用可能**です。
* **圧倒的な高速化**: 本家オープンソース版のWhisperに比べ、同じモデルサイズで**最大4倍高速**に動作し、GPUだけでなくCPUでも高速に文字起こしが行えます。
* **メモリ消費量の削減**: 量子化（int8, float16など）をサポートしており、VRAM（ビデオメモリ）やRAMの消費を劇的に抑えます。
* **選べるモデルサイズ**: `tiny`, `base`, `small`, `medium`, `large-v3` など、PCスペックと要求精度に合わせてモデルを柔軟に選択できます。

### ② 利用方法 (Python例)
`pip install faster-whisper` で導入し、Pythonスクリプトから簡単に呼び出せます。

```python
from faster_whisper import WhisperModel

# モデルのロード (ローカルに自動ダウンロードされます)
# CPUで動かす場合は device="cpu", compute_type="int8"
# GPU(CUDA)で動かす場合は device="cuda", compute_type="float16"
model = WhisperModel("base", device="cpu", compute_type="int8")

# 音声ファイルの認識 (リアルタイムにキャプチャしたWAVデータなども処理可能)
segments, info = model.transcribe("recorded_audio.wav", beam_size=5)

print(f"検知された言語: {info.language} (確信度: {info.language_probability:.2f})")

for segment in segments:
    print(f"[{segment.start:.2f}s -> {segment.end:.2f}s] {segment.text}")
```

### ③ 料金・コスト
* **利用料金**: **完全無料 ($0)**。オープンソースライセンス (MIT License) の下で自由に利用できます。
* **リソースコスト**: ローカルPCのCPU/GPUリソースとメモリを消費します。
  - `tiny` / `base` モデル: CPUで動作し、ゲームプレイへの影響はごく僅か。
  - `small` / `medium` モデル: 精度は高いが、ある程度のVRAM（GPU）またはCPUコアを消費するため、配信中のゲームと干渉する可能性があります。

| 評価軸 | Gemini 1.5/2.0 Flash | GPT-4o mini | Faster-Whisper (ローカル) |
| :--- | :--- | :--- | :--- |
| **コスト** | 🌟 **極めて優秀**<br>- 無料枠が非常に強力。<br>- 有料枠でもGPT-4o-miniの半額。 | ◯ **優秀**<br>- 有料のみだが、従来のGPT-4等と比較すると極めて安価。 | 🌟 **完全無料**<br>- 何時間文字起こしをしてもAPI料金は一切かからない。 |
| **処理速度 (低遅延)** | ◯ **高速**<br>- 実用上十分高速だが、GPT-4o-miniに比べると僅かに遅延があるケースも。 | 🌟 **最速クラス**<br>- APIレスポンスが極めて速く、即時の応答生成に適する。 | 🌟 **極めて高速**<br>- ローカル処理のためネットワーク遅延がなく、モデルサイズ次第でリアルタイム文字起こしが可能。 |
| **コンテキスト容量** | 🌟 **圧倒的 (200万T)**<br>- 数時間の配信音声や、大量のリスナー履歴を余裕で保持。 | ◯ **十分 (128k T)**<br>- 配信中の会話履歴を保持するには実用上必要十分。 | - (STTのため該当なし) |

## 5. エンドユーザー個人アカウントでのLLM利用・認証方法の検討

ローカルで動作するアプリケーションにおいて、開発者側がAPI料金を負担（または中継サーバーを維持）することなく、**「エンドユーザー（配信者個人）自身のアカウントやキー」**を利用して安全に動作させるための具体的なアプローチと実現方法の比較です。

### A. APIキーの直接入力方式 (BYOK: Bring Your Own Key)
ユーザー自身が開発者プラットフォームでAPIキーを発行し、デスクトップアプリの設定画面に直接入力してローカルに保存（暗号化）させる方式です。現在、多くの個人開発ソフトウェアで主流となっています。

* **Gemini (Google AI Studio) での方法**:
  1. ユーザーに「[Google AI Studio](https://aistudio.google.com/)」へ自身のGoogleアカウントでログインしてもらう。
  2. 画面上の「Create API Key」ボタンを1クリックするだけで、無料でAPIキーが即時発行される。
  3. 発行されたキー（`AIzaSy...`）をアプリの設定欄にコピー＆ペーストする。
* **OpenAI (GPT-4o-mini) での方法**:
  1. ユーザーに「OpenAI Platform」でアカウントを作成し、クレジットカードを登録してもらう。
  2. APIキー（`sk-...`）を発行し、アプリに入力する。
* **評価**:
  - **メリット**: 開発者はサーバー代の負担が一切なく、アプリの配布だけで済みます。特にGeminiの場合、ユーザーは**AI Studioの無料枠の範囲内で、完全に無料かつ無制限に近い形でアプリを利用し続けられます**。
  - **デメリット**: ノン技術者の配信者にとって「APIキーを発行してコピペする」という作業が難しそうに感じられる（UX上の心理的ハードル）。

---

### B. OAuth 2.0 認証方式
アプリ上で「Googleアカウントでサインイン」等の認証画面を表示し、ユーザー自身の認証情報を取得してAPIを呼び出す方式です。

* **技術的な仕組み**:
  - 開発者はGoogle Cloud ConsoleでクライアントIDを発行し、アプリにOAuth 2.0の認証フローを組み込みます。
  - ユーザーがログインすると、アプリはユーザーの「アクセストークン（AccessToken）」および「リフレッシュトークン」をローカルに取得します。
* **認証後のAPI利用方法**:
  - 取得したトークンを使用し、**Google Vertex AI（Google Cloudのエンタープライズ向けAIプラットフォーム）** のAPIをユーザーのプロジェクトの課金枠として呼び出します。
* **評価**:
  - **メリット**: APIキーのコピペが不要で、ボタンを押すだけで認証が完了するため、**UX（ユーザー体験）が極めてスムーズで親しみやすい**です。
  - **デメリット**: 非常に設定が複雑です。Google AI Studio（無料枠API）は開発者用のAPIキーによる利用を前提としているため、手軽なOAuth認証だけで「一般ユーザー個人の無料枠」を使わせることは技術的・ポリシー的に困難です。OAuthを使う場合、ユーザー自身にGoogle Cloudプロジェクトを作らせて課金を有効化させる必要があり、かえって導入難易度が跳ね上がります。

---

### C. ローカルLLM (Ollama等) の連携方式（APIキー不要）
ユーザーのPC内に、ローカルでLLMを動かすためのミドルウェア（**Ollama** 等）をインストールしてもらい、アプリからローカルのポート（`http://localhost:11434`）経由で通信する方式です。

* **技術的な仕組み**:
  - アプリのインストーラー、またはマニュアルを通じてOllamaと軽量LLM（`Llama 3 (8B)` や `Gemma 2 (2B)` など）をローカルに導入。
  - アプリはインターネット通信を一切せず、ローカルホストにAPIリクエストを投げて応答を得ます。
* **評価**:
  - **メリット**: **APIキーの発行も、アカウント作成も、クレジットカード登録も一切不要**です。また、完全オフラインで動作するためプライバシーが完璧に守られます。
  - **デメリット**: 配信者が「ゲームをプレイし、配信をエンコードする」傍らで、さらに「ローカルLLMを動かす」ことになるため、GPUやメモリの消費が非常に激しく、高スペックPCを持つ配信者以外では動作が困難になります。
